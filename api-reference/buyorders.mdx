Base URL: `https://api.mannco.store`. All endpoints require **Connected + API** (JWT from `POST /user/login`).

## Create buy order

```
POST /item/buyorder
```

**Permission:** Connected + API

Creates a buy order for a specific item at a specified price. Only one buy order per item per user; use update to change an existing one.

### Body Parameters

<ParamField body="itemid" type="integer" required>
  Item ID for which to create a buy order
</ParamField>

<ParamField body="value" type="integer" required>
  Buy order price in cents (must be positive)
</ParamField>

<ParamField body="amount" type="integer" required>
  Number of items to buy (max 499999 per create; max 4999 per update)
</ParamField>

### Request Example

```json
{
  "itemid": 12345,
  "value": 15000,
  "amount": 3
}
```

### Response

```json Success
{
  "err": false,
  "success": true,
  "content": {
    "details": "Inserted"
  }
}
```

### Error Responses

| Status | Message | Description |
|--------|---------|-------------|
| `error` | Not enough balance | User doesn't have enough balance (value × amount reserved) |
| `error` | You already have buy order on this item please remove them in your profile page! | One buy order per item per user |
| `error` | Max 5000 items | Amount exceeds limit (create: under 500000, update: under 5000) |
| `error` | Value/Amount/Itemid must be a (positive) number | Invalid or missing body parameters |

<Info>
  The total (value × amount) is reserved from your balance when creating a buy order.
</Info>

## Update buy order

```
POST /item/buyorder/update
```

**Permission:** Connected + API

Replaces your existing buy order for the same item with a new price and/or quantity. You must already have a buy order on the item.

### Body Parameters

<ParamField body="itemid" type="integer" required>
  Item ID of the buy order to update
</ParamField>

<ParamField body="value" type="integer" required>
  New price in cents (must be positive)
</ParamField>

<ParamField body="amount" type="integer" required>
  New quantity (max 4999)
</ParamField>

### Request Example

```json
{
  "itemid": 12345,
  "value": 16000,
  "amount": 5
}
```

### Response

```json Success
{
  "err": false,
  "success": true,
  "content": "Buy order updated"
}
```

### Error Responses

| Status | Message | Description |
|--------|---------|-------------|
| `error` | Not enough balance | Insufficient balance for value × amount |
| `error` | You dont have buy order on this item ! | No existing buy order for this item |
| `error` | No buy order found | Remove failed internally |
| `error` | Max 5000 items | Amount exceeds 4999 |

<Warning>
  Updating reserves the new total (value × amount) from your balance; the previous reservation is released.
</Warning>

## Remove buy order

```
POST /item/buyorder/remove
```

**Permission:** Connected + API

Cancels and removes your buy order for an item, releasing the reserved balance.

### Body Parameters

<ParamField body="itemid" type="integer" required>
  Item ID of the buy order to remove
</ParamField>

### Request Example

```json
{
  "itemid": 12345
}
```

### Response

```json Success
{
  "err": false,
  "success": true,
  "content": {
    "details": "Removed"
  }
}
```

### Error Responses

| Status | Message | Description |
|--------|---------|-------------|
| `error` | No buy order found | No buy order for this item for the current user |

## Get user's buy orders for an item

```
GET /user/buyorder/{item}
```

**Permission:** Connected + API

<ParamField path="item" type="string" required>
  Item identifier (numeric ID)
</ParamField>

Returns all active buy orders for a specific item created by the authenticated user.

### Response

```json
{
  "err": false,
  "success": true,
  "content": {
    "informations": {
      "id": 98765,
      "steamid": "76561198000000000",
      "itemid": 12345,
      "price": 15000,
      "amount": 3,
      "timestamp": "1706745600"
    }
  }
}
```

### Buy Order Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | integer | Buy order ID |
| `steamid` | string | User's Steam ID |
| `itemid` | integer | Item ID |
| `price` | integer | Buy order price in cents |
| `amount` | integer | Quantity of items to buy |
| `timestamp` | string | Unix timestamp of creation |

## Get all user's buy orders

```
GET /user/getBuyorder
```

**Permission:** Connected + API

Returns active buy orders for the authenticated user across all items.

### Query Parameters

<ParamField query="page" type="integer" default="0">
  Page number for pagination (offset = page × count)
</ParamField>

<ParamField query="count" type="integer" default="10">
  Number of results per page (max 1000, must be > 0)
</ParamField>

<ParamField query="search" type="string">
  Optional search term to filter by item effect or name
</ParamField>

<ParamField query="undercut" type="boolean">
  If `true` or `1`, filter to buy orders that are below the current lowest sale price
</ParamField>

  "success": true,

### Response Fields

Returns joined data from `buyorder` and `items` tables:

**Buy Order Fields:**
| Field | Type | Description |
|-------|------|-------------|
| `id` | number | Buy order ID |
| `steamid` | string | User's Steam ID |
| `itemid` | number | Item ID |
| `price` | number | Buy order price in cents |
| `amount` | number | Quantity remaining |
| `name` | string | Buy order custom name (or item name) |

**Item Fields (from items table):**
All fields from the `items` table are joined, including `effect`, `name`, `url`, `game`, `quality`, `image`, `type`, `craftable`, etc.

**Count Object:**
| Field | Type | Description |
|-------|------|-------------|
| `count.nb` | number | Total number of buy orders |

<Info>
- Results are ordered by `buyorder.id DESC`
- Only returns buy orders where `amount != 0`
- Search functionality requires at least 3 characters
- The `count` object contains the total number of buy orders for pagination
</Info>

<Info>
  Buy orders are automatically filled when matching items are listed at or below your buy order price. You'll receive a notification when a buy order is filled.
</Info>
