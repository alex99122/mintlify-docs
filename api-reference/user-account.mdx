## Disconnect (Logout)

```
GET /user/disconnect
```

**Permission:** Connected + API

Terminates the current user session and invalidates the JWT token.

### Response

```json
{
  "err": false,
  "success": true,
  "content": {
    "disconnect": true
  }
}
```

---

## Disconnect all sessions

```
GET /user/disconnectSession/all
```

**Permission:** Connected + API

Terminates all active sessions for the authenticated user across all devices.

### Response

```json
{
  "err": false,
  "success": true,
  "content": {
    "message": "All sessions have been disconnected successfully"
  }
}
```

<Info>
  This is useful if you believe your account has been compromised or you want to log out from all devices at once.
</Info>

---

## Disconnect session by ID

```
GET /user/disconnectSession/{id}
```

**Permission:** Connected + API

<ParamField path="id" type="integer" required>
  The session ID to disconnect
</ParamField>

Terminates a specific session by its ID. You can get session IDs from the IP/session list endpoint.

### Response

```json
{
  "err": false,
  "success": true,
  "content": {
    "message": "Session disconnected successfully"
  }
}
```

## Get user information

```
GET /user/infos
```

**Permission:** Connected + API

Returns detailed information about the authenticated user's account.

### Response

```json
{
  "err": false,
  "success": true,
  "content": {
    "informations": {
      "steamId": "76561198000000000",
      "balance": 250000,
      "skrill": "user@example.com",
      "tradeurl": "https://steamcommunity.com/tradeoffer/new/?partner=...",
      "since": "1640000000",
      "image": "https://avatars.steamstatic.com/...",
      "2fa": "true",
      "name": "PlayerName",
      "lastactivity": "1706745600",
      "discordtag": "username#1234",
      "discord": "123456789012345678",
      "shorturl": "customurl",
      "message": "",
      "notification": 1
    }
  }
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `steamId` | string | Steam ID (64-bit) |
| `balance` | number | Account balance in cents |
| `skrill` | string | Registered email address |
| `tradeurl` | string | Steam trade offer URL |
| `since` | string | Account creation timestamp |
| `image` | string | Steam avatar URL |
| `2fa` | string | Two-factor authentication status: `"true"` or `"false"` (string, not boolean) |
| `name` | string | Steam display name |
| `lastactivity` | string | Last activity timestamp |
| `discordtag` | string | Discord tag (username#discriminator) |
| `discord` | string | Discord user ID |
| `shorturl` | string | Custom short URL identifier |
| `message` | string | User bio/message |
| `notification` | number | Notification settings |

<Warning>
Note: The `2fa` field is returned as a string (`"true"` or `"false"`), not a boolean value.
</Warning>

## Get IP / session list

```
GET /user/ipList
```

**Permission:** Connected + API

Retrieves a paginated list of login sessions with IP information, location, and ISP details.

### Query Parameters

<ParamField query="page" type="integer" default="0">
  Page number for pagination
</ParamField>

<ParamField query="perpage" type="integer" default="10">
  Number of results per page (max 50)
</ParamField>

<ParamField query="expire" type="string">
  Set to `"true"` to include expired sessions
</ParamField>

### Response

```json
{
  "err": false,
  "success": true,
  "content": {
    "values": [
      {
        "id": 12345,
        "ip": "192.168.1.1",
        "user": "76561198000000000",
        "location": "US: New York",
        "isp": "Provider Name",
        "current": true,
        "userAgent": "Mozilla/5.0...",
        "timestamp": "1706745600",
        "expire": "1709423600"
      }
    ],
    "count": 25
  }
}
```

<ResponseField name="data.values[].current" type="boolean">
  `true` if this is the current active session
</ResponseField>

## Get notifications

```
GET /user/notifications
```

**Permission:** Connected + API

Returns recent notifications for the authenticated user.

### Query Parameters

<ParamField query="unreadOnly" type="boolean" default="false">
  Return only unread notifications
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Maximum number of notifications to return (max 100)
</ParamField>

### Response

```json
{
  "err": false,
  "success": true,
  "content": {
    "count": 3,
    "paymentReviews": 1
  }
}
```

### Notification Fields

| Field | Type | Description |
|-------|------|-------------|
| `count` | integer | Number of unread notifications/alerts |
| `paymentReviews` | integer | Number of unread payment reviews |
