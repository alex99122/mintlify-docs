Base URL: `https://api.mannco.store`.

Routes here use the **`api` filter**. Endpoints marked **Connected + API** need `Authorization: Bearer <jwt>` from `POST /user/login`. Endpoints marked **API only** do not require a user session (no Bearer).

## Get item details

```
GET /item/details/{item}
```

**Permission:** API only (no session required)

<ParamField path="item" type="string" required>

## Get item sales graph

```
GET /item/salesGraph/{item}?period={period}
```

**Permission:** Connected + API

<ParamField path="item" type="string" required>
  Item identifier (numeric ID)
</ParamField>

<ParamField query="period" type="string" required>
  Time period for sales data: `1M`, `3M`, `6M`, `1Y`, `5Y`, or `ALL`
</ParamField>

Returns historical sales data for charting. Data is aggregated from the `items_stats` table with daily price and volume. For periods `5Y` and `ALL`, data is grouped by week instead of by day.

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `period` | string | Yes | Time period: `1M` (1 month), `3M` (3 months), `6M` (6 months), `1Y` (1 year), `5Y` (5 years), `ALL` (all time) |

### Response (Daily data for 1M, 3M, 6M, 1Y)

```json
{
  "err": false,
  "success": true,
  "content": {
    "values": [
      {
        "date": "2025-01-15 00:00:00",
        "price": 50000,
        "nb": 3
      },
      {
        "date": "2025-01-14 00:00:00",
        "price": 48000,
        "nb": 2
      }
    ]
  }
}
```

### Response (Weekly data for 5Y and ALL)

```json
{
  "err": false,
  "success": true,
  "content": {
    "values": [
      {
        "first_date": "2025-01-13",
        "week": "202502",
        "total_price": 150000,
        "total_nb": 8
      },
      {
        "first_date": "2025-01-06",
        "week": "202501",
        "total_price": 120000,
        "total_nb": 6
      }
    ]
  }
}
```

### Response Fields (Daily)

| Field | Type | Description |
|-------|------|-------------|
| `date` | string | Date and time (YYYY-MM-DD HH:MM:SS) |
| `price` | integer | Total sales value in cents for the day |
| `nb` | integer | Number of sales that day |

### Response Fields (Weekly)

| Field | Type | Description |
|-------|------|-------------|
| `first_date` | string | First day of the week (Monday) |
| `week` | string | Year and week number (YYYYWW) |
| `total_price` | integer | Total sales value in cents for the week |
| `total_nb` | integer | Number of sales that week |

<Info>
  Use daily data (`1M`, `3M`, `6M`, `1Y`) for detailed graphs. Use weekly data (`5Y`, `ALL`) for long-term trends to reduce data points.
</Info>

## Get listing count

```
GET /item/listing/count/{item}
GET /item/listing/count/{item}/{userid}
```

**Permission:** API

<ParamField path="item" type="string" required>
  Item identifier (numeric ID)
</ParamField>

<ParamField path="userid" type="string">
  Optional user SteamID to filter listings by seller
</ParamField>

Returns the number of active listings for an item. If `userid` is provided, returns the count for that specific seller only.

### Response

```json
{
  "err": false,
  "success": true,
  "content": {
    "count": 13291
  }
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `err` | boolean | Error flag (false if no error) |
| `success` | boolean | Success flag (true if successful) |
| `content.count` | integer | Number of active listings for the item |

## Get item listings

```
GET /item/listing/{item}
GET /item/listing/{item}/{userid}
```

**Permission:** API

<ParamField path="item" type="string" required>
  Item identifier (numeric ID)
</ParamField>

<ParamField path="userid" type="string">
  Optional user SteamID to filter by seller
</ParamField>

<ParamField query="count" type="integer" default="10">
  Number of items per page
</ParamField>

<ParamField query="page" type="integer" default="0">
  Page number (offset = page Ă— count)
</ParamField>

<ParamField query="game" type="integer">
  Game ID filter: `440` (TF2), `730` (CS2), `570` (Dota 2), `252490` (Rust), or `753` (Steam). Affects which extra info is joined.
</ParamField>

Returns paginated listings sorted by price ascending. The number of items per page is capped by the server configuration (`maxCount.itemListing`). Each listing includes backpack details. For non-admin callers, `user` and `ip` are omitted from each listing.

### Listing type

| Field | Type | Description |
|-------|------|-------------|
| `id` | integer | Backpack entry ID |
| `assetId` | string | Steam asset ID |
| `item_id` | integer | Item definition ID |
| `user` | string | Seller SteamID |
| `state` | integer | Item state (`1` = on sale) |
| `price` | integer | Listed price in cents |
| `bot` | string | Bot holding the item |
| `game` | integer | Game App ID |
| `wear` | float\|null | Wear value (CS2 items) |
| `sheen` | string | Killstreak sheen (TF2) |
| `killstreaker` | string | Killstreaker effect (TF2) |
| `spell` | string | Halloween spell (TF2) |
| `parts` | string | Strange parts (TF2) |
| `html` | string | Custom HTML description |
| `paint` | string | Paint color (TF2) |
| `values` | object\|null | Extra inspection values (CS2/TF2) |
| `getImage` | string\|null | Custom image URL |

## Get buy order list

```
GET /item/buyorderList/{item}
```

**Permission:** API

<ParamField path="item" type="string" required>
  Item identifier (numeric ID)
</ParamField>

Returns active buy orders for an item, grouped by price tier. Up to 5 price tiers are returned as keys `0`â€“`4`; additional tiers are aggregated under the key `"more"`.

### Response

```json
{
  "err": false,
  "success": true,
  "content": {
    "informations": {
      "0": { "count": 5, "price": 15000 },
      "1": { "count": 3, "price": 14500 },
      "2": { "count": 12, "price": 14000 },
      "more": { "count": 8, "price": 13500 }
    }
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `informations[n].count` | integer | Total quantity at this price |
| `informations[n].price` | integer | Buy order price in cents |
| `informations.more` | object | Aggregated count and lowest price for remaining tiers |

## Get item prices by game

```
GET /item/prices
```

**Permission:** Connected + API

<ParamField query="game" type="integer" required>
  Game App ID: `440` (TF2), `730` (CS2), `570` (Dota 2), `252490` (Rust), or `753` (Steam)
</ParamField>

<ParamField query="outofstock" type="integer">
  Include items with no stock: `0` = in-stock only, `1` = include out-of-stock items
</ParamField>

Returns a list of items for the given game with name, craftable flag, effect, URL slug, in-stock asset count, and lowest sale price.

<Note>
  **Rate limit:** **1 request per 5 minutes** per API key for this endpoint. Exceeding the limit returns HTTP `429` with standard rate-limit headers.
</Note>

### Response

```json
{
  "err": false,
  "success": true,
  "content": [
    {
      "name": "The Scottish Handshake",
      "craftable": 1,
      "effect": "",
      "url": "440-the-scottish-handshake",
      "assetcount": 809,
      "price": 1
    },
    {
      "name": "The Amputator",
      "craftable": 1,
      "effect": "",
      "url": "440-the-amputator",
      "assetcount": 48,
      "price": 1
    }
  ]
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Item display name |
| `craftable` | integer | `1` = craftable, `0` = uncraftable |
| `effect` | string | Unusual effect name (TF2) or empty |
| `url` | string | URL slug (`{game}-{slugified-name}`) |
| `assetcount` | integer | Number of listed assets in stock |
| `price` | integer | Lowest sale price in cents |

### Example

```
GET /item/prices?game=440&outofstock=0
```

## Get item pricing

```
GET /item/pricing/{item}
```

**Permission:** Connected + API

<ParamField path="item" type="string" required>
  Item identifier (numeric ID)
</ParamField>

Returns calculated pricing data for an item including lowest sale, highest buy order, Steam price, suggested listing price, and optional last sale.

### Response

```json
{
  "err": false,
  "success": true,
  "content": {
    "item_id": 12345,
    "cached": true,
    "last_updated": "2025-02-19 12:00:00",
    "pricing": {
      "lowest_sale_price": 15000,
      "lowest_buy_order": 12000,
      "steam_price": 18000,
      "suggested_price": 14999,
      "last_sale": {
        "price": 14800,
        "date": 1706745600
      }
    }
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `item_id` | integer | Item ID |
| `cached` | boolean | Whether the result was served from cache |
| `last_updated` | string | Last calculation or cache update (Y-m-d H:i:s) |
| `pricing.lowest_sale_price` | float | Lowest active listing price in cents |
| `pricing.lowest_buy_order` | float | Highest active buy order price in cents |
| `pricing.steam_price` | float | Steam Community Market price in cents |
| `pricing.suggested_price` | float | Suggested listing price |
| `pricing.last_sale` | object/null | Last sale price and date (Unix timestamp) if available |

<Info>
  Pricing data is cached for 20 minutes. Fresh data is calculated and stored when the cache expires.
</Info>

## Get bulk pricing

```
GET /item/pricing/bulk
```

**Permission:** Connected + API

<ParamField query="items" type="string" required>
  Comma-separated item IDs (maximum 100 items)
</ParamField>

Returns pricing data for multiple items at once. Each entry mirrors `GET /item/pricing/{item}` pricing fields, plus **`from_cache`**: `true` if the row was read from `prices` while still considered fresh, `false` if prices were recalculated and written back to `prices`.

The envelope includes **`cached_items`** and **`refreshed_items`** counts (same semantics as `from_cache`, summed over every returned row â€” duplicate IDs in the query string are counted separately).

### Response

```json
{
  "err": false,
  "success": true,
  "content": {
    "total_items": 3,
    "cached_items": 2,
    "refreshed_items": 1,
    "items": [
      {
        "item_id": 12345,
        "from_cache": true,
        "last_updated": "2025-02-19 12:00:00",
        "pricing": {
          "lowest_sale_price": 15000,
          "lowest_buy_order": 12000,
          "steam_price": 18000,
          "suggested_price": 14999,
          "last_sale": { "price": 14800, "date": 1706745600 }
        }
      },
      {
        "item_id": 67890,
        "from_cache": false,
        "last_updated": "2025-02-19 12:01:05",
        "pricing": {
          "lowest_sale_price": 9000,
          "lowest_buy_order": 8500,
          "steam_price": 9500,
          "suggested_price": 8999,
          "last_sale": null
        }
      }
    ]
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `total_items` | integer | Number of item entries returned (same length as `items`) |
| `cached_items` | integer | How many entries were served from cache (`from_cache` true) |
| `refreshed_items` | integer | How many entries were recalculated and persisted (`from_cache` false) |
| `items[].item_id` | integer | Item ID |
| `items[].from_cache` | boolean | `true` if read from `prices` without recalculation; `false` if refreshed this request |
| `items[].last_updated` | string | `updated_at` from DB when cached, or time of refresh (`Y-m-d H:i:s`) |
| `items[].pricing.*` | | Same as [Get item pricing](#get-item-pricing) (`lowest_sale_price`, `lowest_buy_order`, `steam_price`, `suggested_price`, `last_sale`) |

<Info>
  Bulk pricing reuses rows in `prices` when `updated_at` is within the last **5 minute** (shorter than the single-item `GET /item/pricing/{item}` cache window). Outside that window, the API recomputes from live tables and updates `prices`.
</Info>

### Example

```
GET /item/pricing/bulk?items=12345,67890,11111
```

## Get item details from backpack ID (CS2)

```
GET /item/cs/details/fromid/{backpackid}
```

**Permission:** API

<ParamField path="backpackid" type="string" required>
  Backpack asset ID
</ParamField>

Same as above but specifically for CS2 items with wear, sticker, and pattern information.

### Response Fields for Backpack Details

Both endpoints return combined data from `backpack`, `items`, and `item_infos` tables:

```json
{
  "err": false,
  "success": true,
  "content": {
    "informations": {
      // Backpack fields
      "id": "integer",
      "assetId": "string",
      "user": "string (Steam ID)",
      "item_id": "integer",
      "price": "integer (cents) or null",
      "state": "integer (0=inventory, 1=on sale, 2=withdrawal)",
      "bot": "string (bot Steam ID)",
      "game": "integer (440/730/570/252490/753)",
      "wear": "float or null (CS2 items)",
      "sheen": "string (TF2 killstreak)",
      "killstreaker": "string (TF2 killstreak)",
      "spell": "string (TF2 spells)",
      "parts": "string or null (TF2 strange parts)",
      "paint": "string or null (TF2 paint)",
      "lastupdate": "integer (Unix timestamp)",
      
      // Items table fields
      "name": "string",
      "effect": "string",
      "quality": "string",
      "type": "string",
      "craftable": "integer or empty string",
      "image": "string",
      "rarity": "string",
      "url": "string",
      "color": "string (hex color)",
      
      // item_infos fields (if available)
      "values": "string (JSON) or null",
      "getImage": "integer or null"

```

<Info>
**Important:** The `craftable` field is set to empty string (`""`) for all non-TF2 games. Only TF2 items (game 440) retain the original `craftable` value (0 or 1). Additionally, trailing spaces and trailing semicolons are removed from all string fields.
</Info>
