---
title: "Vendor webhooks"
description: "POST /webhook/vendor/{provider} for marketplace vendor integrations"
---

```
POST https://api.mannco.store/webhook/vendor/{provider}
```

**Permission:** Route includes the **`api`** filter (programmatic / integration use). This endpoint is intended for **vendor backends** that receive callbacks from external marketplaces, not for typical end-user scripts.

<ParamField path="provider" type="string" required>
  Vendor implementation id resolved by `VendorFactory` on the server.
</ParamField>

## Request body

JSON or form-encoded payload as produced by the upstream vendor. If the body is empty, the server tries to parse JSON from `php://input`.

## Processing

The handler forwards **POST data** and **HTTP headers** to the provider-specific `handleWebhook` implementation and returns:

```json
{
  "err": false,
  "success": true,
  "content": {}
}
```

where `content` is the **vendor-specific result object** (structure depends on integration).

## Errors

| content | When |
|---------|------|
| Provider is required | Missing `provider` in path |
| Unknown vendor provider | Provider not registered |
| Invalid JSON | Body present but not valid JSON |
| No POST data | Empty body after parse |

<Info>
  Configure your vendor integration in the Mannco.store server environment; this documentation does not list provider-specific signatures or header rules.
</Info>
