## Process Cashout

<Card title="POST /cashout/process" icon="money-bill-transfer">
  Initiates a cashout request. Validates the user's balance, cashout parameters, and deducts the amount before creating the cashout record.
</Card>

```
POST /cashout/process
```

**Authentication:** `connected`

**Body Parameters**

<ParamField body="type" type="string" required>
  The cashout method type (e.g., `"paypal"`, `"payoneer"`, `"tipalti"`).
</ParamField>

<ParamField body="amount" type="integer" required>
  The amount to cash out (in cents).
</ParamField>

<ParamField body="provider" type="string" required>
  The payment provider (`"paypal"`, `"payoneer"`, or `"tipalti"`).
</ParamField>

**Response**

```json
{
  "status": "success",
  "data": {
    "message": "Cashout processed successfully",
    "cashout_id": 12345,
    "amount": 10000,
    "type": "paypal",
    "status": "pending"
  }
}
```

**Errors**

| Status | Message |
|--------|---------|
| `error` | User not authenticated |
| `error` | Missing required parameters: type and amount |
| `error` | Insufficient balance |
| `error` | Invalid amount |
| `error` | Failed to deduct balance |

## Link Cashout Method

<Card title="POST /cashout/link" icon="link">
  Links a cashout method to the user's account. Supports PayPal, Payoneer, and Tipalti with different flows for each provider.
</Card>

```
POST /cashout/link
```

**Authentication:** `connected`

**Body Parameters (all providers)**

<ParamField body="provider" type="string" required>
  The payment provider: `"paypal"`, `"payoneer"`, or `"tipalti"`.
</ParamField>

### PayPal & Payoneer

Returns a link URL that the user should follow to complete account linking.

### Tipalti — Additional Parameters

<ParamField body="payeeType" type="string" default="Individual">
  `"Individual"` or `"Company"`.
</ParamField>

<ParamField body="firstName" type="string" required>
  First name (required for Individual payee).
</ParamField>

<ParamField body="lastName" type="string" required>
  Last name (required for Individual payee).
</ParamField>

<ParamField body="companyName" type="string">
  Company name (required for Company payee type).
</ParamField>

<ParamField body="email" type="string" required>
  Email address for the payee.
</ParamField>

<ParamField body="address" type="object" required>
  Address object with the following fields:
  <Expandable title="Address fields">
    <ParamField body="address.address1" type="string" required>Street address line 1.</ParamField>
    <ParamField body="address.address2" type="string">Street address line 2.</ParamField>
    <ParamField body="address.city" type="string" required>City name.</ParamField>
    <ParamField body="address.postalCode" type="string" required>Postal/ZIP code.</ParamField>
    <ParamField body="address.country" type="string" required>ISO country code.</ParamField>
    <ParamField body="address.state" type="string">State/province (optional).</ParamField>
  </Expandable>
</ParamField>

<ParamField body="phone" type="string">
  Phone number (optional).
</ParamField>

<ParamField body="taxDetails" type="object">
  Tax details (optional).
  <Expandable title="Tax fields">
    <ParamField body="taxDetails.taxpayerId" type="string">Taxpayer ID.</ParamField>
    <ParamField body="taxDetails.taxFormType" type="string">Tax form type.</ParamField>
  </Expandable>
</ParamField>

<ParamField body="paymentMethod" type="object">
  Payment method preferences (optional — can be set during onboarding).
  <Expandable title="Payment method fields">
    <ParamField body="paymentMethod.paymentMethodName" type="string">Payment method name.</ParamField>
    <ParamField body="paymentMethod.paymentDetails" type="object">Payment method details.</ParamField>
  </Expandable>
</ParamField>

### Response — PayPal / Payoneer

```json
{
  "status": "success",
  "data": {
    "message": "Cashout link created successfully",
    "link": "https://paypal.com/connect/...",
    "provider": "paypal",
    "method": "link"
  }
}
```

### Response — Tipalti

```json
{
  "status": "success",
  "data": {
    "message": "Tipalti payee account created successfully",
    "provider": "tipalti",
    "payeeId": "payee_abc123",
    "alreadyExists": false
  }
}
```

## Unlink Cashout Method

<Card title="POST /cashout/unlink" icon="link-slash">
  Removes a linked cashout method from the user's account. Currently only supports Payoneer.
</Card>

```
POST /cashout/unlink
```

**Authentication:** `connected`

**Body Parameters**

<ParamField body="provider" type="string" required>
  The provider to unlink. Currently only `"payoneer"` is supported.
</ParamField>

**Response**

```json
{
  "status": "success",
  "data": {
    "message": "Payoneer cashout method unlinked successfully",
    "provider": "payoneer"
  }
}
```

**Errors**

| Status | Message |
|--------|---------|
| `error` | Missing required parameter: provider |
| `error` | Invalid provider |
| `error` | Failed to unlink cashout method. The method may not be linked or an error occurred. |
