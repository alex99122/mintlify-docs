## Get deposit information

```
GET /deposit/{game}
```

**Permission:** Connected + API

<ParamField path="game" type="integer" required>
  Game ID (`440` for TF2, `730` for CS2, `570` for Dota 2, `252490` for Rust)
</ParamField>

Returns pricing information and available items from the user's Steam inventory for the specified game.

### Response

```json
{
  "status": "success",
  "data": {
    "items": [
      {
        "assetId": "123456789",
        "classId": "987654321",
        "instanceId": "0",
        "name": "Unusual Burning Flames Team Captain",
        "marketName": "Unusual Team Captain",
        "icon": "https://...",
        "marketable": 1,
        "tradable": 1,
        "price": 150000,
        "suggested_price": 145000,
        "accepted": true
      }
    ],
    "steamInventoryUrl": "https://steamcommunity.com/profiles/.../inventory",
    "totalValue": 450000
  }
}
```

### Item Fields

| Field | Type | Description |
|-------|------|-------------|
| `assetId` | string | Steam asset ID |
| `classId` | string | Steam class ID |
| `name` | string | Item display name |
| `marketName` | string | Steam Market name |
| `price` | integer | Mannco.store buy price in cents |
| `suggested_price` | integer | Suggested listing price in cents |
| `accepted` | boolean | Whether the item is accepted for deposit |
| `tradable` | integer | 1 if tradable, 0 if trade locked |
| `marketable` | integer | 1 if marketable on Steam |

<Info>
  Only items that are tradable, marketable, and have a valid price will have `accepted: true`.
</Info>

## Get bot list

```
GET /deposit/botList/{game}
```

**Permission:** API

<ParamField path="game" type="integer" required>
  Game ID
</ParamField>

Returns a list of available trade bots for the specified game, including their status and capacity.

### Response

```json
{
  "status": "success",
  "data": [
    {
      "steamId": "76561198000000000",
      "name": "Mannco Bot #1",
      "online": true,
      "capacity": 850,
      "currentItems": 645,
      "tradeUrl": "https://steamcommunity.com/tradeoffer/new/?partner=..."
    }
  ]
}
```

## Get trade status

```
GET /deposit/tradeStatus/{tradeid}
```

**Permission:** Connected + API

<ParamField path="tradeid" type="string" required>
  Steam trade offer ID
</ParamField>

Checks the current status of a deposit trade offer.

### Response

```json
{
  "status": "success",
  "data": {
    "tradeId": "987654321",
    "status": 3,
    "statusText": "Completed",
    "items": [
      {
        "assetId": "123456789",
        "name": "Unusual Team Captain",
        "price": 150000
      }
    ],
    "totalPrice": 150000,
    "timestamp": "1706745600",
    "completedAt": "1706749200"
  }
}
```

### Trade Status Values

| Status | Description |
|--------|-------------|
| `0` | Pending - waiting for acceptance |
| `3` | Completed - items credited to account |
| `-1` | Failed - trade was declined or expired |

## Set price for upcoming trade

```
POST /deposit/setPriceForUpcomingTrade
```

**Permission:** Connected + API

Pre-sets custom prices for items before creating a deposit trade. Useful for negotiating prices or handling items with dynamic pricing.

### Body Parameters

<ParamField body="items" type="array" required>
  Array of items with assetId and custom price
</ParamField>

### Request Example

```json
{
  "items": [
    {
      "assetId": "123456789",
      "price": 145000
    },
    {
      "assetId": "987654321",
      "price": 85000
    }
  ]
}
```

### Response

```json Success
{
  "status": "success",
  "data": {
    "pricesSet": 2,
    "expiresAt": "1706749200",
    "message": "Prices set for upcoming trade. Create trade within 10 minutes."
  }
}
```

<Warning>
  Custom prices expire after 10 minutes. You must initiate the deposit trade within this timeframe.
</Warning>

<Info>
  This endpoint is typically used in conjunction with a manual price negotiation process. Most users will use the standard pricing from `GET /deposit/{game}`.
</Info>
