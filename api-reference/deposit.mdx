Endpoints use **Connected + API** unless noted. Send `Authorization: Bearer <jwt>`.

## Get deposit information

```
GET /deposit/{game}
```

**Permission:** Connected + API

<ParamField path="game" type="integer" required>
  Game id: `440` (TF2), `730` (CS2), `570` (Dota 2), `252490` (Rust).
</ParamField>

Returns your Steam inventory for that game with **`depositkey`** maps per asset id, pricing hints, and optional `disabled` / high-stock fields. The response uses `content.informations` as an array of rows.

### Response (shape)

```json
{
  "err": false,
  "success": true,
  "content": {
    "informations": [
      {
        "assetid": "123456789;987654321",
        "count": 2,
        "market_hash_name": "Unusual Burning Flames Team Captain",
        "item_id": 5678,
        "url": "unusual-burning-flames-team-captain",
        "depositkey": {
          "123456789": "a1b2c3d4e5f6...",
          "987654321": "f6e5d4c3b2a1..."
        },
        "nb_high_stock": 45,
        "high_stock_limit": 200
      }
    ]
  }
}
```

TF2 / CS2 / Dota / Rust may add game-specific fields (`unusual`, `wear`, `stickers`, etc.). See existing **Items** docs for field meanings.

<Info>
  Each `depositkey[assetId]` is validated server-side with `hash('sha256', depositSecret + assetId + user)` when you submit **POST /deposit/trade**.
</Info>

## Create deposit trade

```
POST /deposit/trade
```

**Permission:** Connected + API

### Body parameters

<ParamField body="prices" type="object" required>
  Map **assetId string** to price in **cents** (integer). Can be sent as JSON object or JSON string parsed server-side.
</ParamField>

<ParamField body="depositKeys" type="object" required>
  Map **assetId** to the **depositkey** string from `GET /deposit/{game}` for that asset.
</ParamField>

<ParamField body="game" type="integer">
  Default `440`. Must be one of `440`, `730`, `570`, `252490`.
</ParamField>

### Example

```json
{
  "prices": { "123456789": 15000, "987654321": 8000 },
  "depositKeys": { "123456789": "...", "987654321": "..." },
  "game": 440
}
```

### Response

```json
{
  "err": false,
  "success": true,
  "content": { "id": 42 }
}
```

Use `content.id` as **`tradeid`** for **GET /deposit/tradeStatus/{tradeid}**.

### Errors (typical)

| content message | Meaning |
|-----------------|---------|
| missing value parameter | `prices` missing |
| missing depositKeys parameter | `depositKeys` missing |
| invalid game parameter | Bad or missing `game` |
| invalid JSON format in value parameter | `prices` not valid JSON when sent as string |
| invalid JSON format in depositKeys parameter | Same for keys |
| value must be a JSON object with assetId - price mapping | `prices` not an object |
| invalid deposit key for assetId: X | Hash mismatch |
| failed to save deposit price | Server could not persist |

## Instant sell (information)

```
GET /deposit/instantSell/{game}
```

**Permission:** Connected + API

<ParamField path="game" type="integer" required>
  Currently **440** (TF2) only in server implementation.
</ParamField>

Returns enriched inventory data for instant-sell pricing (`content.informations`), built from cached deposit lists and buy-order tiers.

---

## Create instant sell trade

```
POST /deposit/trade/instant
```

**Permission:** Connected + API (+ `2fa` filter â€” satisfied automatically with API JWT)

Same general idea as **POST /deposit/trade**, plus:

<ParamField body="cashout_method" type="string" required>
  Instant sell payout method (server-defined values).
</ParamField>

**Supported games:** TF2 (`440`) only in code.

Also requires `prices` and `depositKeys` as objects (or JSON strings). Errors mirror validation in `createTradeInstant.php` (`missing prices parameter`, `instant sell is not supported for this game`, etc.).

## Get trade status

```
GET /deposit/tradeStatus/{tradeid}
```

**Permission:** Connected + API

<ParamField path="tradeid" type="integer" required>
  The **`id`** returned by **POST /deposit/trade** (deposit row id).
</ParamField>

Returns deposit row plus nested **`trade`** when available.

### Response (shape)

```json
{
  "err": false,
  "success": true,
  "content": {
    "trade": {
      "id": 123456,
      "items_received": "123456789,987654321",
      "items_send": "",
      "status": 3,
      "user": "76561198000000000",
      "bot": "76561198111111111",
      "code": "...",
      "lasterror": "",
      "offerid": "5678901234",
      "timestamp": "1706745600000",
      "game": 440
    }
  }
}
```

### Errors

| Type | When |
|------|------|
| `error` | Missing or invalid `tradeid` |
| `notFound` | No deposit for that id |
| `forbidden` | Deposit belongs to another user |

### Trade status (deposit flow)

| Status | Meaning |
|--------|---------|
| `0` | Pending |
| `3` | Completed |
| `-1` | Failed |
