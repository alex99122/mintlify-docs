## Get Buy Order & Lowest Sale Prices

<Card title="GET /infos/prices/buyorderAndLowestSales" icon="tags">
  Retrieves a global list of all items with their highest buy order price and lowest sale price. Data is cached for performance and can be force-refreshed.
</Card>

```
GET /infos/prices/buyorderAndLowestSales
```

**Authentication:** None

**Query Parameters**

<ParamField query="update" type="string">
  If present, bypasses the cache and fetches fresh data.
</ParamField>

**Response**

```json
{
  "status": "success",
  "data": {
    "prices": [
      {
        "item_id": 1,
        "buyorder_price": 1500,
        "lowest_sale_price": 2000
      },
      {
        "item_id": 2,
        "buyorder_price": 500,
        "lowest_sale_price": 750
      }
    ]
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `item_id` | integer | Unique item type identifier |
| `buyorder_price` | integer | Highest active buy order price (in cents) |
| `lowest_sale_price` | integer | Lowest current sale listing price (in cents) |

<Note>
  Prices are in cents (divide by 100 for dollar amounts). This endpoint is heavily cached â€” use the `update` parameter only when necessary.
</Note>

## Get Admin Info

<Card title="GET /infos/admin" icon="user-shield">
  Retrieves the list of admin/staff members. Requires API key authentication.
</Card>

```
GET /infos/admin
```

**Authentication:** `api` (API key required), `noDB`

**Response**

Returns a JSON array of staff member information.

```json
[
  {
    "name": "AdminName",
    "steamId": "76561198000000000",
    "role": "admin"
  }
]
```
